## FSD 폴더 구조 도입

### 배경
- SvelteKit의 `src/lib/`에 FSD(Feature-Sliced Design) 라이트 구조를 적용하기로 결정
- `src/routes/`가 FSD의 pages/app 레이어를 이미 담당하므로, `src/lib/` 안에는 4개 레이어만 배치

### 구조

```
src/lib/
├── shared/          ← 공용 (utils, assets, paraglide, ui)
├── entities/        ← 비즈니스 엔티티 (product, user 등)
├── features/        ← 유저 액션 단위 (cart 등)
└── widgets/         ← 조합 블록 (product-list 등)
```

### import 방향 규칙

```
shared ← entities ← features ← widgets ← routes
```

상위 → 하위만 허용. 역방향 import 시 ESLint 에러.

## eslint-plugin-boundaries 설정

### 설치한 패키지
- `eslint-plugin-boundaries` — 계층 간 import 방향 강제
- `eslint-import-resolver-typescript` — `$lib` 별칭 해석 (tsconfig paths 활용)

### eslint.config.js에 추가한 설정

```js
{
  files: ['src/**/*.{ts,js,svelte}'],
  plugins: { boundaries },
  settings: {
    'boundaries/elements': [
      { type: 'shared',   pattern: ['src/lib/shared/*'],   mode: 'folder' },
      { type: 'entities', pattern: ['src/lib/entities/*'], mode: 'folder' },
      { type: 'features', pattern: ['src/lib/features/*'], mode: 'folder' },
      { type: 'widgets',  pattern: ['src/lib/widgets/*'],  mode: 'folder' },
      { type: 'routes',   pattern: ['src/routes/*'],       mode: 'folder' },
    ],
    'import/resolver': {
      typescript: { alwaysTryTypes: true }
    }
  },
  rules: {
    'boundaries/element-types': ['error', {
      default: 'disallow',
      rules: [
        { from: 'shared',   allow: ['shared'] },
        { from: 'entities', allow: ['shared', 'entities'] },
        { from: 'features', allow: ['shared', 'entities'] },
        { from: 'widgets',  allow: ['shared', 'entities', 'features', 'widgets'] },
        { from: 'routes',   allow: ['shared', 'entities', 'features', 'widgets'] },
      ]
    }]
  }
}
```

### 주의사항
- `$lib` 별칭을 boundaries가 인식하려면 import 대상 파일이 실제로 존재해야 함
- `eslint-import-resolver-alias`는 동작하지 않았고, `eslint-import-resolver-typescript`가 tsconfig의 `paths` 설정을 읽어서 해결

## 기존 lib 파일 재배치

### 이동한 폴더
| 이전 경로 | 이후 경로 |
|-----------|-----------|
| `src/lib/assets/` | `src/lib/shared/assets/` |
| `src/lib/paraglide/` | `src/lib/shared/paraglide/` |

### 수정한 파일 (5개)
| 파일 | 변경 내용 |
|------|-----------|
| `vite.config.ts` | `outdir: './src/lib/shared/paraglide'` |

| `src/hooks.ts` | `$lib/shared/paraglide/runtime` |
| `src/routes/+layout.svelte` | `$lib/shared/assets/...`, `$lib/shared/paraglide/...` |
| `src/routes/demo/paraglide/+page.svelte` | `$lib/shared/paraglide/...` |

## 메모: SvelteKit 폴더/import 정리

- `src/lib/` → `$lib` 별칭 자동 제공 (tsconfig에서 설정 불필요)
- `.svelte` 파일은 그 자체가 default export — `export default` 문 불필요
- `.ts` 유틸/타입은 일반 `export` 사용
- `src/lib/index.ts`에서 re-export하면 `$lib`로 짧게 import 가능 (배럴 패턴)
