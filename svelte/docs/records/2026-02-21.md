## 결정: Tailwind CSS 미사용 (초기 셋업)

### 배경
- Svelte 5는 `<style>` 태그가 자동 scoped CSS로 동작
- React에서는 CSS 캡슐화가 없어서 Tailwind + 래퍼 컴포넌트 패턴이 유용했음
- Svelte에서는 그 패턴이 불필요 — scoped CSS가 기본 제공

### 결정
- 초기 셋업에서 Tailwind CSS를 사용하지 않음
- Svelte의 scoped CSS로 스타일링

### 근거
- 용량 차이 없음 (둘 다 사용한 것만 번들에 포함)
- Svelte 학습 단계에서 scoped CSS를 직접 써봐야 Svelte 스타일링 철학을 체감할 수 있음
- 나중에 필요하면 `npx sv add tailwindcss` 한 줄로 추가 가능

- `sv add mcp`는 불필요 (플러그인이 동일 기능 제공)

## 메모: React vs Svelte DX 차이

- React: CRA 죽은 뒤 Vite + ESLint + Prettier + Testing Library 등 직접 조합해야 함. Next.js도 테스트/스토리북은 수동 셋업
- Svelte: `sv create` 하나로 공식 검증된 애드온 조합을 한 번에 셋업. "batteries included" 철학
- 프레임워크 차원에서 tooling 추천 → 호환성 문제 없음, 셋업 시간 절약
